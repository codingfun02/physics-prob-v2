1. 전체 그림 — 30초 요약

우리 프로그램은 두 단계로 나뉩니다.

┌─────────────────────────────────────────────────────────────┐
│  [설계 단계]  연속 밀도 ρ(x,y,z)  →  격자  →  강체 물성 계산   │
│               (컴퓨터가 적분을 '더하기'로 근사)                  │
│   출력: M, COM, I₁,I₂,I₃, 주축 방향                           │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  [시뮬 단계]  PyBullet이 '정육면체 강체 1개'를 떨어뜨림         │
│               (1728칸을 따로 시뮬하지 않음!)                    │
│   입력: 물성 + 랜덤 자세 + 랜덤 각속도 + DROP_HEIGHT           │
│   출력: 바닥에 닿은 면 번호 (1~6)                             │
└─────────────────────────────────────────────────────────────┘
                              ↓
                    N번 반복 → 면별 확률 p_k

핵심 한 줄:

격자는 PyBullet 안에서 매 순간 쓰이는 게 아니라,

「이 정육면체 안에 질량이 어떻게 몰려 있는지」를 숫자로 바꿔 PyBullet에 한 번 넘기기 위한 도구입니다.

1.1 우리가 구하는 것

주사위를 $H=6$ m 높이에서 떨어뜨린 뒤, 바닥에 닿은 면이 1~6 중 무엇인지의 확률 $p_k$입니다.

$$p_k = P(\text{바닥의 면} = k)$$

위쪽 면이 아니라 바닥 면을 셉니다 (get_bottom_face()). 마주보는 면은 합이 7입니다 (바닥이 2면이면 위는 5면).

3. 밀도 함수 ρ(x,y,z)란?

3.1 수학적 정의

주사위가 차지하는 영역을 $\Omega$라 하면, 각 점 $(x,y,z)$마다 밀도 $\rho(x,y,z)$가 있습니다.

$$\rho(x,y,z) \geq 0$$

작은 부피 $\Delta V$ 안의 질량:

$$\Delta m \approx \rho(x,y,z) \cdot \Delta V$$

3.2 우리 프로젝트의 좌표계

config.py 기준:

  • 주사위 한 변 길이: $L = 1\,\mathrm{m}$
  • 중심이 원점 $(0,0,0)$
  • 각 축 범위: $[-0.5,\,0.5]$
        z (+)
        ↑
        |   ┌──────┐  윗면 = 눈 1 (+z)
        |  /      /|
        | └──────┘ │
        |/       / |
   ─────┼───────┼──→ x (+)  오른쪽 = 눈 2
       /|       |  |
      / └───────┘  |
     /             |
    y (+) 앞쪽 = 눈 3

3.3 코드에서의 밀도 함수 예시

density/analytic.py에 연속 함수로 정의합니다.

균일 (공정한 주사위):

$$\rho = 1 \quad \text{(어디나 같음)}$$

구형 bump (변인 통제 실험):

$$\rho(x,y,z) = \begin{cases} f & \text{구 내부} \\ 1 & \text{구 바깥} \end{cases}$$

구 내부 조건 ($x_0, y_0, z_0$ 중심, 반지름 $R$):

$$(x-x_0)^2 + (y-y_0)^2 + (z-z_0)^2 \leq R^2$$

코드 (rho_sphere_bump):

r2 = (x - x0) ** 2 + (y - y0) ** 2 + (z - z0) ** 2
if r2 <= radius**2:
    return factor   # 예: 4.0 또는 5.0
return 1.0

3.4 밀도가 확률에 영향을 주는 경로

밀도 $\rho$는 직접 “몇 번 면이 나온다”고 정해 주지 않습니다.

$$\rho \;\Rightarrow\; M,\;\mathbf{r}_{\mathrm{cm}},\;\mathbf{I} \;\Rightarrow\; \text{낙하·충돌 동역학} \;\Rightarrow\; \text{최종 자세} \;\Rightarrow\; \text{바닥의 면}$$

즉 밀도는 질량 분포를 정하고, 그게 운동 방정식을 바꿉니다.

4. 12×12×12 격자 — 왜 나누는가?

4.1 컴퓨터는 연속 함수를 그대로 못 씀

연속 $\rho(x,y,z)$는 이론상 무한히 많은 점이 필요합니다.

컴퓨터는 유한 개 칸에 나눠 저장합니다.

$$\rho_{ijk} \approx \rho(x_i,\, y_j,\, z_k)$$

  • $i,j,k = 0,1,\ldots,n-1$
  • 현재 $n = 12$ → $12^3 = 1728$칸

4.2 한 칸(소셀)의 크기

$$\Delta = \frac{L}{n} = \frac{1}{12} \approx 0.0833\,\mathrm{m}$$

한 칸 부피:

$$V_{\mathrm{cell}} = \Delta^3 = \left(\frac{1}{12}\right)^3 \approx 5.79 \times 10^{-4}\,\mathrm{m}^3$$

한 칸 질량:

$$m_{ijk} = \rho_{ijk} \cdot V_{\mathrm{cell}}$$

4.3 격자 중심 좌표

각 칸의 대표 점은 칸 중심입니다 (density/grid.py):

$$x_i = -0.5 + \left(i + \tfrac{1}{2}\right)\Delta$$

discretize_rho()는 이 중심에서 $\rho$를 읽습니다:

for i, x in enumerate(axis):
    for j, y in enumerate(axis):
        for k, z in enumerate(axis):
            rho[i, j, k] = func(x, y, z)

4.4 6×6×6이 아니라 12×12×12인 이유

해상도칸 수장단점
$6^3=216$적음빠르지만, 안쪽 구 bump가 뭉개져 COM·관성이 부정확
$12^3=1728$중간bump 형태 보존, COM·관성 오차 감소 (현재 GRID_N=12)
$24^3=13824$많음더 정확하지만 계산·메모리 증가

비유: 사진을 픽셀 많이 찍을수록 둥근 공이 더 둥글게 보이는 것과 같습니다.

격자가 거칠면 “구 bump”가 계단 모양이 되어, 질량중심이 잘못 잡힙니다.

4.5 격자가 PyBullet 시뮬 안에서 쓰이지 않음

이 점이 가장 헷갈리기 쉬운 부분입니다.

  • 격자 역할: $M$, COM, $\mathbf{I}$ 계산용 (설계 단계)
  • PyBullet 역할: 상자 1개 강체로 충돌·낙하 계산 (시뮬 단계)

1728칸을 각각 PyBullet body로 만들지 않습니다.

5. 격자에서 질량·질량중심·관성 구하기

physics/inertia.py의 compute_inertia()가 하는 일입니다.

5.1 총 질량 $M$

연속:

$$M = \int_\Omega \rho(\mathbf{r})\, dV$$

격자 근사 (리만 합):

$$M = \sum_{i,j,k} \rho_{ijk}\, V_{\mathrm{cell}}$$

코드:

masses = (grid.rho * grid.cell_volume).reshape(-1)
total_mass = float(np.sum(masses))

5.2 질량중심 $\mathbf{r}_{\mathrm{cm}}$

연속:

$$\mathbf{r}_{\mathrm{cm}} = \frac{1}{M} \int_\Omega \rho(\mathbf{r})\,\mathbf{r}\, dV$$

격자:

$$\mathbf{r}_{\mathrm{cm}} = \frac{\sum_{i,j,k} m_{ijk}\,\mathbf{r}_{ijk}}{M}$$

여기서 $\mathbf{r}_{ijk} = (x_i, y_j, z_k)$는 칸 중심 좌표.

코드:

com = np.sum(masses[:, np.newaxis] * centers, axis=0) / total_mass

예 (ctrl_factor_f50, 12³ 격자):

$\mathbf{r}_{\mathrm{cm}} \approx (0.031,\, 0,\, 0)$ — +x 쪽으로 약 3 cm 치우침.

5.3 왜 질량중심이 중요한가?

원점과 질량중심이 다르면:

  1. 중력 토크 — 같은 자세라도 “넘어지려는 방향”이 달라짐
  2. 충돌 시 지레-arm — 바닥 반력이 COM과 다른 위치에서 작용 → 회전 유발
  3. 위치 에너지 — 면이 바닥일 때 COM 높이 $z_{\mathrm{cm}}$이 자세마다 다름

중력 토크 (COM 기준):

$$\boldsymbol{\tau}_{\mathrm{grav}} = \mathbf{r}_{\mathrm{cm}} \times (M\mathbf{g})$$

$\mathbf{g} = (0,0,-g)$이면, COM이 원점에서 벗어난 만큼 기울어지려는 힘이 생깁니다.

6. 관성텐서와 주관성 모멘트 3개

6.1 점질량의 관성 모멘트 (복습)

질량 $m$인 점이 회전축에서 거리 $d$에 있으면:

$$I = m d^2$$

멀리 있을수록 돌리기 어렵습니다.

6.2 3×3 관성텐서 $\mathbf{I}$

한 축만이 아니라 3차원 공간에서는 방향마다 돌리기 어려움이 다릅니다.

이를 행렬 $\mathbf{I}$로 씁니다.

COM 기준, 각 소셀 $m$, 위치 $\mathbf{r}=(x,y,z)$에 대해 누적:

$$I_{xx} = \sum m(y^2 + z^2), \quad I_{yy} = \sum m(x^2 + z^2), \quad I_{zz} = \sum m(x^2 + y^2)$$

$$I_{xy} = -\sum m\,xy, \quad I_{xz} = -\sum m\,xz, \quad I_{yz} = -\sum m\,yz$$

코드 (physics/inertia.py):

I[0, 0] += m * (y * y + z * z)   # I_xx
I[1, 1] += m * (x * x + z * z)   # I_yy
I[2, 2] += m * (x * x + y * y)   # I_zz
I[0, 1] -= m * x * y             # I_xy
...

6.3 주축과 주관성 모멘트 $I_1, I_2, I_3$

$\mathbf{I}$는 대칭 행렬이라 고유값 분해가 가능합니다:

$$\mathbf{I}\,\mathbf{e}_k = I_k\,\mathbf{e}_k, \quad k=1,2,3$$

  • $I_1 \leq I_2 \leq I_3$: 주관성 모멘트 (3개 숫자)
  • $\mathbf{e}_1,\mathbf{e}_2,\mathbf{e}_3$: 주관성 축 방향

물리적 의미: 주축을 중심으로 도는 회전은 다른 축 성분을 “끌어당기지” 않아서 가장 단순합니다.

코드:

eigvals, eigvecs = eigh(I)
principal_moments = eigvals[order]   # [I1, I2, I3]
principal_axes = eigvecs[:, order]   # 주축 방향
rot = Rotation.from_matrix(principal_axes)

6.4 “관성 모멘트 3개만”이 전부인가?

아닙니다. PyBullet에 필요한 것:

넘기는 값개수PyBullet 인자
총 질량 $M$1baseMass
질량중심3baseInertialFramePosition
주관성 모멘트3localInertiaDiagonal
주축 방향 (회전)4 (쿼터니언)baseInertialFrameOrientation
충돌 모양상자 1개GEOM_BOX

합치면 스칼라 1 + 벡터 3 + 대각 3 + 쿼터니언 4 + 마찰·반발 등.

7. PyBullet에 실제로 넘기는 값들

physics/bodies.py의 create_die():

col = p.createCollisionShape(p.GEOM_BOX, halfExtents=[0.5, 0.5, 0.5])

die_id = p.createMultiBody(
    baseMass=props.mass,
    baseCollisionShapeIndex=col,
    baseInertialFramePosition=props.com.tolist(),
    baseInertialFrameOrientation=principal_quat,
)

p.changeDynamics(
    die_id, -1,
    lateralFriction=FRICTION,          # 0.5
    restitution=RESTITUTION,           # 0.75 (현재)
    localInertiaDiagonal=props.principal_moments.tolist(),
)

7.2 왜 상자 1개인가?

실제 주사위도 겉모양은 정육면체입니다.

안쪽만 무겁게 만든 “치트 주사위”와 같습니다.

PyBullet은 강체 역학이 잘 검증된 엔진이라,

복잡한 내부 구조를 1728개 body로 쪼개지 않고 등가 강체 1개로 처리합니다.

8. 초기 랜덤 조건 — 수학과 코드

simulation/single_trial.py — 매 시행마다 다른 초기 조건.

8.1 위치

$$\mathbf{x}_0 = (0,\; 0,\; H), \quad H = \texttt{DROP\_HEIGHT} = 6\,\mathrm{m}$$

수평 방향은 항상 중앙. 높이만 정해져 있고, 옆으로 던지지 않습니다.

8.2 선속도

$$\mathbf{v}_0 = (0,\; 0,\; 0)$$

처음에는 정지 상태에서 떨어집니다 (손에서 던진 속도 없음).

8.3 자세 — SO(3) 균일 랜덤

3D 회전은 3개 각도로 표현하기 어렵습니다 (짐벌 락).

코드는 단위 쿼터니언을 균일하게 뽑습니다:

q = np.random.randn(4)   # 4개 표준정규 난수
q /= np.linalg.norm(q)   # 길이 1로 정규화

의미: 모든 방향으로 기울어진 자세가 같은 확률로 나옵니다.

8.4 각속도 — 축마다 균일

$$\omega_x, \omega_y, \omega_z \sim \mathrm{Uniform}(-\omega_{\max},\, +\omega_{\max})$$

$\omega_{\max} = \texttt{MAX\_ANGULAR\_VEL} = 2.0\,\mathrm{rad/s}$

w = np.random.uniform(-MAX_ANGULAR_VEL, MAX_ANGULAR_VEL, size=3)

의미: 처음부터 빙글빙글 돌고 떨어집니다.

공정한 주사위를 “손가락으로 튕겨 굴리는” 느낌에 가깝습니다.

8.5 왜 랜덤인가?

만약 항상 같은 자세·같은 각속도로 떨어뜨리면 확률이 아니라 한 가지 결과만 나옵니다.

우리가 원하는 $p_k$는 초기 조건을 무작위로 했을 때의 비율:

$$p_k \approx \frac{\text{면 } k \text{이 바닥인 횟수}}{N}$$

9.3 주사위 body 생성 — create_die()

격자 1728칸을 PyBullet body로 만들지 않습니다. 사전 계산한 강체 물성만 넘깁니다.

col = p.createCollisionShape(p.GEOM_BOX, halfExtents=[0.5, 0.5, 0.5])

die_id = p.createMultiBody(
    baseMass=props.mass,
    baseCollisionShapeIndex=col,
    basePosition=[0, 0, 0],
    baseOrientation=[0, 0, 0, 1],
    baseInertialFramePosition=props.com.tolist(),
    baseInertialFrameOrientation=principal_quat,
)
p.changeDynamics(die_id, -1,
    lateralFriction=FRICTION,
    restitution=RESTITUTION,
    localInertiaDiagonal=props.principal_moments.tolist(),
)
  • 충돌 모양: body 좌표계에서 항상 대칭 정육면체 ($1\times1\times1$ m)
  • 질량 분포: COM이 치우쳐 있고, 주관성 축이 body 축과 약간 다를 수 있음
  • 면 번호: body 축 기준 (FACE_LABELS) — 주관성 축과 별개

9.5 시간 적분 루프 — step_until_settled()

$$\Delta t = \frac{1}{240}\,\mathrm{s} \approx 4.17\,\mathrm{ms}$$

매 스텝 (p.stepSimulation())에서 PyBullet이 하는 일 (개념):

  1. 중력으로 선속도·각속도 갱신
  2. 위치·자세 적분 ($\mathbf{x} \leftarrow \mathbf{x} + \mathbf{v}\Delta t$ 등)
  3. 바닥($z=0$)과 정육면체 충돌 검사
  4. 충돌 시 법선 반력·마찰·반발 적용
  5. 다음 $\Delta t$로

왜 1/240초인가? $H=6$ m 낙하 시 충돌 직전 속도 $\approx 10.8$ m/s이면, 한 스텝 이동 $\approx 4.5$ cm — 주사위(1 m)에 비해 충분히 작아 관통 오류를 줄입니다.

9.6 충돌 — 마찰·반발

바닥·주사위 모두 changeDynamics로 동일한 $\mu=0.5$, $e=0.75$를 씁니다.

법선 방향 (바닥에 수직):

$$v_{\text{충돌 후, 법선}} \approx -e \times v_{\text{충돌 전, 법선}}$$

$e=0.75$ (v3)이면 수직 속도의 75%가 반대 방향으로 — 높이 튕김.

접선 방향: 마찰 $\mu$가 미끄러짐을 줄여 굴러가다 멈추게 합니다.

여러 번의 바운스·굴림이 누적되어, “정적 최소 에너지 자세”와 다른 최종 면이 나올 수 있습니다.

9.7 정지 판정

speed = |v| < 0.01 m/s      # VEL_THRESHOLD
spin  = |ω| < 0.05 rad/s    # ANG_VEL_THRESHOLD

위 조건이 0.1초 연속 (SETTLE_TIME, 24스텝) 유지되면 루프 종료.

최대 2400스텝 ≈ 10초 (MAX_SIM_STEPS)까지 진행하며, 그 안에 멈추지 않으면 그 시점 자세로 면을 판정합니다.

9.8 면 판정 및 시행 종료

주의: 시행마다 connect→disconnect를 새로 합니다. 워커 프로세스는 격자·관성(props)만 미리 계산해 재사용합니다 (monte_carlo._init_worker).

9.9 운동 에너지 관점 (직관)

에너지 종류대략 크기 (v3)

낙하·충돌 에너지가 위치 에너지 차이(면 간 COM 높이 차)보다 훨씬 큽니다.

그래서 “가장 낮은 COM 자세”만으로 결정되지 않고, 동역학(회전·튕김·마찰)이 최종 면을 좌우합니다.

9.10 질량중심이 회전을 바꾸는 이유

COM이 원점에서 벗어나면 중력이 회전(토크)을 만듭니다:

$$\boldsymbol{\tau}_{\mathrm{grav}} = \mathbf{r}_{\mathrm{cm}} \times (M\mathbf{g})$$

회전 운동 방정식:

$$\mathbf{I}\,\dot{\boldsymbol{\omega}} = \boldsymbol{\tau}_{\mathrm{합}}$$

균일 주사위($\mathbf{r}_{\mathrm{cm}}=\mathbf{0}$)에서는 자세 대칭이 남지만, +x 쪽에 무거운 bump가 있으면 자세마다 토크·위치에너지가 달라져 면별 확률이 1/6에서 벗어납니다.

10. 멈춘 뒤 — 바닥의 면 판정

10.1 원리

각 면에는 바깥을 가리키는 법선 벡터 $\mathbf{n}$이 있습니다.

면 번호법선 (body 좌표)
1$(0,0,+1)$
6$(0,0,-1)$
2$(+1,0,0)$
5$(-1,0,0)$
3$(0,+1,0)$
4$(0,-1,0)$

주사위가 회전 행렬 $R$만큼 돌아가 있으면, 월드 좌표 법선:

$$\mathbf{n}_{\mathrm{world}} = R\,\mathbf{n}_{\mathrm{body}}$$

바닥은 월드 $-\hat{z}$ 방향.

$\mathbf{n}_{\mathrm{world}}$가 $(0,0,-1)$에 가장 가깝게 정렬된 면이 바닥:

$$k^* = \arg\max_k \;\mathbf{n}_k \cdot (0,0,-1)$$

코드:

down = np.array([0.0, 0.0, -1.0])
dot = np.dot(world_normal, down)
# dot이 최대인 면의 번호 반환

10.2 “위쪽 면”이 아니라 “바닥 면”

이 프로젝트는 바닥에 닿은 면을 셉니다 (extra.face = "bottom").

표준 주사위에서 바닥이 2번이면 위는 5번입니다 (마주보는 면 합 = 7).

11. 몬테카를로 — 확률 추정

simulation/monte_carlo.py:

# 워커 초기화 시 격자·관성 1회 계산 (시행마다 재계산 안 함)
_init_worker(grid)  # → compute_inertia(grid)

for face in pool.imap(_run_one_trial, range(N)):
    counts[face] += 1

p_k = counts[k] / N

진짜 확률 $p_k$는 이론값이고, 시뮬 결과 $\hat{p}_k$는 비율 추정치입니다:

$$\hat{p}_k = \frac{C_k}{N}$$

11.1 왜 매번 결과가 다른가?

초기 자세·각속도가 매 시행마다 랜덤이기 때문입니다. $N=200$이면 14%–20%처럼 들쭉날쭉해 보여도 정상입니다.

11.2 표준오차

이항분포 근사:

$$\mathrm{SE}(p_k) \approx \sqrt{\frac{p_k(1-p_k)}{N}}$$

$p_k \approx 1/6$일 때 $\mathrm{SE} \approx \sqrt{5/(36N)}$.

시행 횟수 $N$대략 SE의미 (95% 구간 대략)
2002.6%p±5%p 정도 흔들림
5,0000.5%p±1%p
50,0000.17%p±0.3%p
100,000 (v3 실험)0.12%p±0.2%p
500,0000.05%p±0.1%p

규칙: $N$을 4배 늘리면 SE는 약 절반이 됩니다 ($\propto 1/\sqrt{N}$).

차트 오차막대는 $\sqrt{\hat{p}(1-\hat{p})/N}$ — 위 SE의 추정치입니다.

11.3 통계적 오차 vs 체계적 오차

종류뜻$N$을 늘리면?
통계적 오차이번 표본에서만 운이 좋거나 나쁨줄어듦
체계적 오차 (bias)모델·근사 자체의 한계안 줄어듦

체계적 오차 예: 12³ 격자가 거칠어 COM·관성이 부정확, $\Delta t$가 커서 충돌 계산 오차, 정지 판정 기준의 애매함, 등가 강체 1개 근사.

12. 낙하·충돌 파라미터 — 왜 중요한가?

12.1 DROP_HEIGHT $H$

낙하 속도 (끝에서, 공기저항 무시):

$$v \approx \sqrt{2gH} \approx \sqrt{2 \times 9.81 \times 6} \approx 10.8\,\mathrm{m/s}$$

$H$대략 $v$구분효과

13. 자주 하는 오해·FAQ

13.1 오해 정리

오해실제
“격자 1728개를 PyBullet이 매 프레임 계산한다”X — 격자는 사전 계산용, PyBullet은 상자 1개
“관성 모멘트 3개만 넣으면 된다”△ — $M$, COM(3), 주축 쿼터니언(4)도 필요
“+x 무거우면 무조건 2번 바닥”X — 정적으론 2번이 유리해도, 낙하+회전+튕김이면 5번이 많을 수 있음
“2번과 5번 면 라벨이 뒤바뀌었을 것”X — +x=2, −x=5로 일관; 조건별로 2/5 우세 방향이 바뀜 (f30 vs f50 등)
“밀도가 크면 그 면 확률이 바로 커진다”X — $\rho \to$ 물성 $\to$ 전체 동역학 $\to$ 확률
“위쪽 면을 센다”X — get_bottom_face()로 바닥 면 기록
“6×6×6이면 충분하다”△ — 구 bump 실험에선 12³가 더 안정적 (GRID_N=12)

13.2 자주 묻는 질문

Q. 주사위 크기 1 m면 숫자가 이상하지 않나?

오히려 반대입니다. 1 m면 질량·관성이 커서 PyBullet이 다루기 안전합니다. 실제 2 cm 주사위가 수치적으로 더 까다로울 수 있습니다.

Q. 균일 밀도면 정말 1/6인가?

이론적으로 거의 대칭입니다. $N=100{,}000$이면 각 면이 16.5%–16.8% 근처에 모입니다. 200번만 하면 14%–20%처럼 보여도 표본이 적어서 그렇습니다.

Q. PyBullet 내부를 알아야 하나?

아니요. “중력·충돌·마찰을 대신 계산해 주는 엔진”으로 이해하면 됩니다. 우리가 신경 쓰는 것은 입력(밀도, 초기 조건, $H$, $e$, $\mu$)과 출력(바닥 면)입니다.

부록 B — 균일 정육면체 관성 (검증용)

밀도 $\rho = 1$, 부피 $V=1$, 질량 $M=1$인 균일 정육면체:

$$I_{xx} = I_{yy} = I_{zz} = \frac{M L^2}{12} = \frac{1}{12} \approx 0.0833\,\mathrm{kg \cdot m^2}$$

step2_test_inertia.py로 균일 격자에서 이 값에 가까운지 확인합니다.

부록 C — 수식 요약 (복습용)

질량·질량중심:

$$M \approx \sum \rho_{ijk}\,\Delta V, \qquad \mathbf{r}_{\mathrm{cm}} \approx \frac{1}{M}\sum \rho_{ijk}\,\mathbf{r}_{ijk}\,\Delta V$$

중력 토크·위치 에너지:

$$\boldsymbol{\tau} = \mathbf{r}_{\mathrm{cm}} \times (M\mathbf{g}), \qquad U = M g\, z_{\mathrm{cm}}$$

확률 추정:

$$\hat{p}_k = \frac{C_k}{N}, \qquad \mathrm{SE} \approx \sqrt{\frac{p_k(1-p_k)}{N}}$$
